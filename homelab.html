Home Lab Infrastructure

Proxmox-based home lab built to host self-managed services, test deployment workflows, and build practical skills in systems administration, networking, automation, and monitoring. Functions as both a working environment and a technical learning platform.

Architecture

System Overview

Multiple containers and services on Proxmox, each separated by role to improve maintainability, troubleshooting, and resource control.

⚙️

Virtualization Layer

Proxmox as the primary platform for running and managing lightweight containers and services from a single host machine.

🔒

Service Isolation

Applications split across separate containers so updates, failures, and configuration changes remain isolated from each other.

📈

Operational Focus

Lab used to practice deployment, storage planning, network config, remote access, performance tuning, and troubleshooting.

Stack

Services Running in Proxmox

Major services hosted in the environment and the role each one serves inside the lab.

Jellyfin Media Server

Self-hosted media organization and streaming. Built experience with storage mapping, permissions, remote access, and hardware transcoding.

Media Automation Stack

Radarr, Sonarr, NZBGet, and Seerr working together to automate media requests, acquisition, and organization. Required handling service communication, shared mounts, and application permissions.

n8n Automation

Automation workflows for real tasks — parsing structured data, moving information between services, and reducing repetitive manual work.

Grafana & Prometheus

Prometheus collects metrics; Grafana visualizes performance data. Together they provide visibility into CPU, memory, service behavior, and trends over time.

Samba / Shared Storage

Shared storage exposed through managed mounts — separates compute from storage while giving containers controlled access to media and data directories.

Reverse Proxy & Network Services

Experimentation with network access, internal service routing, remote connectivity, and segmentation between management and application traffic.

Takeaways

Real-World Applications & Lessons Learned

This project turned abstract IT and systems concepts into repeatable, hands-on work.

  • Linux administration and service configuration
  • Storage design, mount points, and file permissions
  • Network troubleshooting and service accessibility
  • Monitoring, logging, and performance analysis
  • Incremental testing and fault isolation

Professional Relevance

  • Mirrors real infrastructure tasks in IT and systems roles
  • Demonstrates experience with uptime, reliability, and maintenance
  • Shows ability to document, debug, and improve multi-service environments
  • Builds familiarity with tools used in self-hosting, DevOps, and platform ops
  • Provides concrete examples for interviews and technical discussions

What This Project Taught

Infrastructure is not just about launching services. It requires planning, dependency management, storage layout decisions, reliable networking, and continuous troubleshooting. Operating this environment built practical confidence in diagnosing failures, tracing resource issues, organizing services by role, and improving systems over time rather than treating deployment as a one-time task.
