SocialMind — AI Content Studio

Full-stack SaaS platform (Postle) for AI-generated social media content. Node.js backend with SQLite, JWT auth, and OpenAI integration — deployed on AWS EC2 behind nginx with GitHub Actions CI/CD. Includes a complete dashboard UI for managing and generating posts.

Architecture

How It's Built

Full-stack application from frontend to cloud deployment — every layer built and configured manually.

🖥️

Frontend

Single-page dashboard with auth gate, content generation UI, post management, and real-time feedback — served as static files via nginx.

⚙️

Backend

Node.js + Express REST API with JWT authentication, SQLite for persistence, and OpenAI API integration for AI content generation.

☁️

Infrastructure

AWS EC2 (Ubuntu, t3.small) with nginx reverse proxy, PM2 process management, Let's Encrypt TLS, and automated deploys via GitHub Actions.

Tech Stack

Technologies Used

Each technology chosen for a specific role in the deployment pipeline.

Node.js / Express REST API server handling auth, content requests, and data persistence
OpenAI API AI content generation for social media posts based on user prompts and platform targets
SQLite Lightweight embedded database for user accounts, sessions, and generated content
JWT Auth Token-based authentication with protected routes on both frontend and API
AWS EC2 Ubuntu 22.04 instance running the full stack — t3.small for cost-effective hosting
nginx Reverse proxy routing traffic to Node port 3000, serving static frontend, and handling HTTPS termination
PM2 Process manager keeping Node server alive across reboots with ecosystem config
GitHub Actions CI/CD pipeline — auto-deploys to EC2 via SSH on push to main branch
Capabilities

Features & What Was Built

End-to-end product covering content generation, auth, and deployment.

Application Features

  • AI-powered post generation targeting specific platforms and tones
  • User authentication with JWT — protected dashboard and API routes
  • Post history and management in the admin panel
  • Responsive single-page UI with real-time generation feedback
  • Admin panel for monitoring usage and managing content

Infrastructure Work

  • EC2 instance provisioned and bootstrapped from scratch with shell script
  • nginx configured as reverse proxy with HTTPS via Let's Encrypt / certbot
  • PM2 ecosystem config for process management and auto-restart
  • GitHub Actions workflow for zero-downtime deploys over SSH
  • Security group rules scoped to required ports only

What This Project Taught

Building Postle end-to-end meant owning every layer — API design, auth flows, AI integration, database schema, cloud provisioning, and deployment automation. The infrastructure side was particularly valuable: configuring nginx, setting up TLS, managing processes with PM2, and wiring up CI/CD gave hands-on experience with the full production deployment lifecycle that goes beyond just writing application code.
